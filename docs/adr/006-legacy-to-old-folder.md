# ADR 006: Легаси-код в `old/`

**Status:** Accepted
**Date:** 2026-04-14

## Context

Первая версия проекта (Laravel 11 + Breeze + CRUD без реализованного бронирования) полностью не соответствует новой архитектуре (DDD, модульный монолит, Filament, Vue 3 SPA). Нужно решить:
1. Удалить старый код
2. Оставить в git history, рабочая директория чистая
3. Переместить в `old/` для справки

## Decision

**Переместить в `old/`.** Не удалять. Не коммитить `old/vendor/`, `old/node_modules/` (в .gitignore).

## Consequences

**Плюсы:**
- Легко сослаться на старую миграцию/конфиг
- Не требует копаться в git history
- Понятная граница старое/новое

**Минусы:**
- Немного шума в файловой системе
- Риск случайного изменения в old/

**Mitigation:** CI можно научить падать при изменениях в `old/`.

## Alternatives considered

- **Удалить** — отвергнут: возможно понадобится сослаться на старые миграции
- **Git tag старой версии + удалить** — отвергнут: менее удобно для чтения
