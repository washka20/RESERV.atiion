# ADR 008: Без Event Sourcing на старте

**Status:** Accepted
**Date:** 2026-04-14

## Context

Event Sourcing (ES) — хранить историю как последовательность событий, state восстанавливается replay'ем. Для финтеха/trading — must have. Для букинга?

## Decision

**Без ES.** Domain Events + таблица `domain_events` для аудита — достаточно.

## Consequences

**Плюсы:**
- Проще разработка (в 3-5 раз)
- Меньше зависимостей
- Легче порт в Go (ES-библиотеки различаются)

**Минусы:**
- Нельзя восстановить произвольное историческое состояние
- Ограничения на аудит (только то, что явно залогировали)

**Mitigation:** таблица `domain_events` даёт сырые события для аудита.

## Alternatives considered

- **Полный ES (spatie/laravel-event-sourcing)** — отвергнут: избыточно, крутая кривая входа
- **ES только для Payment** — отложено: добавим, если понадобится, точечно
